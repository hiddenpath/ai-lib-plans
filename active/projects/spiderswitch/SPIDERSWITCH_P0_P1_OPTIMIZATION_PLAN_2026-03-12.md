# spiderswitch P0/P1 优化执行计划（2026-03-12）

## 1. 背景与目标

基于本轮代码审查，`spiderswitch` 当前存在以下高优先级问题：

- P0-1：初始化路径存在阻塞式网络 I/O，可能拉高 MCP 首次调用延迟
- P0-2：切换模型时修改全局环境变量（代理），存在并发污染风险
- P1-1：异常回包透传内部异常文本，存在信息泄露面
- P1-2：模型 ID 冲突会静默覆盖，缺乏冲突可观测性

本计划目标：

1. 将 P0/P1 问题在本轮迭代内完成代码级修复与回归验证  
2. 把 P2 项目纳入后续任务治理与迭代队列  
3. 形成可追踪、可回滚、可验收的执行记录

## 2. 任务分层与范围

### P0（本轮必须完成）

- **P0-A 初始化去阻塞化**
  - 从运行时请求路径中移除默认官方 dist 同步
  - 将同步动作改为显式触发（函数/CLI），避免工具首包被网络阻塞

- **P0-B 并发安全**
  - 移除模型切换路径中对 `os.environ` 的临时改写
  - 改为“检测并报错/提示”模式，不对进程级环境做副作用修改

### P1（本轮必须完成）

- **P1-A 错误响应收敛**
  - 对外统一返回安全消息与错误码
  - 保留详细异常在服务端日志中（脱敏后）

- **P1-B 清单冲突可观测**
  - 检测 `provider/model` 冲突并抛出结构化错误
  - 在错误详情中明确冲突 ID 与来源文件，便于定位

### P2（纳入后续迭代，不在本轮强制）

- 引入 list/status 结果缓存与 TTL 策略
- 增加离线初始化性能基准与并发压测基线
- 扩展端到端 MCP 回归（真实客户端）

## 3. 交付物

- 代码改动：`spiderswitch/src/spiderswitch/**`
- 测试补充：`spiderswitch/tests/**`
- 文档更新：`spiderswitch/README.md`、`README_CN.md`（如行为有变化）
- 验证记录：`pytest`、`ruff`、`mypy` 执行结果

## 4. 验收标准

- P0：
  - 首次 `list_models`/`switch_model` 不再触发默认网络下载行为
  - 切换流程不再改写全局环境变量
- P1：
  - 未知/内部错误响应含 `error.code`，且不透传原始异常文本
  - 模型 ID 冲突能被显式检测并给出结构化错误
- 工程质量：
  - `pytest` 通过
  - `ruff` 通过
  - `mypy` 通过

## 5. 风险与回滚

- 风险：
  - 变更初始化流程后，部分依赖“自动同步 dist”的环境可能需手动同步
  - 代理策略从“临时规避”改为“显式失败”后，可能暴露既有环境配置问题
- 回滚策略：
  - 按文件级回滚 `python_runtime.py` 与工具响应改造
  - 保留新增测试，便于验证回滚后行为差异

## 6. 执行节奏

1. 安装本机缺失工具：`pytest`/`ruff`/`mypy`  
2. 实施 P0（初始化与并发安全）  
3. 实施 P1（错误治理与冲突可观测）  
4. 补测与静态检查，更新任务状态与完成记录
