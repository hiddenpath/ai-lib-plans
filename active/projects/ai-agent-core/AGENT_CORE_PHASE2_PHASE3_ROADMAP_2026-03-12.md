# ai-agent-core Phase2/Phase3 里程碑与验收

日期：2026-03-12  
范围：`ai-agent-core` v0.2.x ~ v0.3.x  
关联：`AGENT_CORE_SCOPE_CONTRACT_2026-03-12.md`

## Phase 2（可扩展）：多任务调度与恢复能力

### 目标
- 从单任务串行闭环升级为可并发的任务编排系统。
- 引入任务持久化与检查点恢复，确保异常中断后可恢复。

### 关键交付
1. 多任务队列与并发 worker（可配置并发度）。
2. 可持久化状态存储（sqlite 优先）：
   - task 表
   - state_transition 表
   - retry_log 表
3. 检查点恢复：
   - 进程重启后恢复 `planned/routed/executing` 中断任务
4. 策略升级：
   - 成本优先策略（cost-first）
   - 质量优先策略（quality-first）
5. 失败治理：
   - error_class 分类统计
   - 路由降级（runtime/model fallback）策略

### 验收标准
- 100 个任务并发压测下，无任务丢失（最终状态可追踪）。
- 在故障注入（进程重启）后，恢复率 >= 99%。
- 至少两种策略可切换，且策略决策记录可审计。
- 与 `spiderswitch` MCP 工具语义保持兼容，无破坏性接口改动。

## Phase 3（可运营）：策略插件化与生产观测

### 目标
- 建立可运营的策略平台能力，支持灰度、A/B 路由和策略插件化。
- 提供生产可观测与治理闭环（SLO、成本、失败率、回退路径）。

### 关键交付
1. 策略插件接口（Policy Plugin API）：
   - 输入：任务上下文、runtime profile、历史指标
   - 输出：route decision + confidence + reason
2. 实验框架：
   - A/B 策略路由
   - 灰度比例控制与快速回滚
3. 观测体系：
   - p50/p95 延迟
   - 成本/任务
   - 成功率与重试率
   - 降级触发率
4. 治理能力：
   - 策略变更审计日志
   - 租户级限额与优先级策略
   - SLO 违规自动降级

### 验收标准
- A/B 策略可在同一任务类型下稳定运行并输出对比报告。
- 可在 5 分钟内完成策略回滚（演练验证）。
- 关键指标可被导出（Prometheus/JSON report 任一）。
- 生产演练中连续 7 天无高优先级不可恢复故障。

## 依赖与风险

### 依赖
- `spiderswitch` 执行面稳定（MCP 工具契约保持）。
- `ai-lib-*` runtime profile 字段持续一致（[ARCH-003]）。
- `ai-protocol` manifest 与能力声明稳定演进（[ARCH-001]）。

### 主要风险与缓解
- 风险：策略复杂度过快膨胀  
  缓解：先固定策略模板，再引入插件机制。
- 风险：执行层与策略层边界漂移  
  缓解：每次版本评审执行契约检查清单。
- 风险：观测指标定义不统一  
  缓解：先冻结指标字典，再推进 dashboard。
